# v0.1.53 Unity Material Corridor Validation

Generated by a Unity 6000.4.6f1 batchmode sidecar at `AssetPacks/BrassworksBreach.UnityMaterialCorridorValidation09`.
This is a true in-engine lookdev scene built from Unity primitives, generated meshes, Unity lights, Unity cameras, Set08 material-role references, and Unity-loaded Set08 texture availability checks. The visible pass uses a sidecar replacement shader with vertex material colors and procedural grain because the first batchmode material shader path correctly failed the magenta-output gate. No Blender, external DCC, Python/PIL image rendering, or flat documentation board renderer was used.

## Result

- Recommendation: `accept`.
- Material-binding guidance: Good enough to guide v0.1.53 selective material placement and corridor composition for floors, wall panels, pipes, gauges, pressure doors, and amber gaslights; revise before treating this as final texture/shader fidelity.
- Passed gates: 8 / 8 PNGs nonblank and below magenta shader-error threshold.

## What Passed

- Wet black stone, black iron panels, worn brass, oxidized copper, amber glass, red pressure enamel, scorched metal, gauge enamel, grime overlays, and oily floor accents all render in a unified steampunk corridor context as material-role proxies.
- The warm/cool lighting comparison shows the materials remain readable under amber gaslight and cooler fill conditions.
- Geometry scale reads correctly for a first-person corridor: 5.5 meter width envelope, low ceiling, floor tile rhythm, wall panel rivets, pipe clusters, door wheel, gauges, and gaslights.

## Limitations / Follow-up

- This is lookdev evidence, not import-ready gameplay geometry; the meshes are validation primitives.
- This is not a final texture-fidelity pass. The sidecar loaded Set08 texture PNGs for availability checks, then rendered visible surfaces with vertex material colors/procedural grain to keep the Unity camera pass non-magenta and inspectable.
- Final gauge glass, gaslight glass, steam, grime, and soot should move to project-decided transparent/decal/emissive shaders before broad production use.
- No baked GI, post-processing stack, animated steam, decals, nav/collision, LODs, occlusion, or platform performance profiling are included in this sidecar.
- The final game should bind these materials selectively rather than flooding every corridor surface with high-detail textures.
