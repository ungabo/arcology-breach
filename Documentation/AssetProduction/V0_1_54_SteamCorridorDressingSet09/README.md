# Steam Corridor Dressing Set 09 Asset Production

## Goal

Create a self-contained Unity sidecar package for a whole steampunk corridor dressing family. The package is designed for fast parallel development without modifying the main project manifest, scenes, or production assets.

## Delivered Package

Package root:

`AssetPacks/BrassworksBreach.SteamCorridorDressingSet09`

The package uses a deterministic Unity editor generator. It ships catalog data and generator code, then lets Unity create the actual materials, meshes, and prefabs in `Runtime/Generated`.

## Look Target

- Blackened stone and soot-stained brick.
- Wet floor plates, drains, and inspection hatches.
- Brass and copper pipework with oily iron clamps.
- Pressure gauges with ivory faces and red needles.
- Amber gaslight lamp lenses in iron/brass cages.
- Valves, steam vents, handrails, cable brackets, ceiling pipe clusters, and doorway pressure-lock dressing.

## Deterministic Authoring Notes

- No Blender or external DCC assets were used.
- No prefab YAML was hand-authored.
- Meshes are generated by Unity editor code from fixed primitive recipes.
- Prefabs are composed from catalog transforms, generated meshes, and generated materials.
- A contact sheet can be reproduced from the metadata JSON with `Tools/GenerateContactSheet.ps1`.

## Integration Boundary

This sidecar is visual-only. Generated prefabs should be treated as art dressing and promoted through quarantine review before any collision, lighting, gameplay, VFX, or audio behavior is added by the main game integration owner.
