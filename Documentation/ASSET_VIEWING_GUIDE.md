# Asset Viewing Guide

Updated: 2026-05-23 21:47 -04:00

This guide explains what the current project assets are, why many neighboring files end in `.meta`, and how to view the available art previews.

## Quick Answer

The `.meta` files are not the assets. They are Unity sidecar metadata files.

For every imported asset and many folders, Unity creates a matching `.meta` file that stores the asset GUID, importer settings, labels, and other bookkeeping. Unity uses those GUIDs to keep scene references stable even if a file is renamed or moved inside the Unity Project window.

Keep `.meta` files in source control. Do not try to view them as art, models, textures, or materials. The real asset is the file next to the `.meta` file, such as `.png`, `.mat`, `.asset`, or `.unity`.

## Why Some Textures Are `.asset` Files

Four current texture files are Unity `Texture2D` assets serialized directly as `.asset` files:

- `Assets/_Project/Textures/T_Steam_BrassHazardPipe.asset`
- `Assets/_Project/Textures/T_Steam_BrassPipe.asset`
- `Assets/_Project/Textures/T_Steam_OilDarkStone.asset`
- `Assets/_Project/Textures/T_Steam_RivetedIron.asset`

These are real Unity texture assets, but they are not normal source PNG files. They were generated by Unity editor code and saved as serialized Unity objects. Unity can read them normally in the editor. Outside Unity, they are awkward to preview, so the preview PNGs in `Documentation/AssetPreviews` reconstruct their visible color data for convenience.

For a final asset pack, the better long-term format is to keep source texture maps as normal image files, such as PNG, TGA, TIFF, or EXR, plus Unity materials/prefabs that reference them. The current `.asset` textures are fine for the prototype.

## Procedural Geometry Versus Imported Assets

Most current visual objects are not imported model files yet. The scene builder creates them from Unity primitives, materials, and scripts.

Examples include:

- corridors, walls, floors, gates, lifts, pipes, gauges, valves, and signs
- the pressure pistol viewmodel
- the steam scattergun pickup
- Scrapper, Lancer, Bulwark, Bellows Node, and Governor Warden placeholder bodies
- pickup objects such as health vials, cartridges, gear keys, and secret caches

That means there is no `.fbx`, `.obj`, or `.prefab` file to double-click for most objects yet. To inspect those objects, open the relevant scene in Unity and select objects in the Hierarchy.

## How To View In Unity

1. Open Unity Hub.
2. Open the project at `D:\__MY APPS\Unity Doom`.
3. In the Project window, browse to `Assets/_Project`.
4. Use these folders first:
   - `ConceptArt`
   - `Materials`
   - `Textures`
   - `Data`
   - `Scenes`
5. Select a `.png`, `.mat`, `.asset`, or `.unity` file to see its Inspector preview.
6. Use the Project search box for quick filtering:
   - `t:Material`
   - `t:Texture2D`
   - `t:Scene`
   - `t:ScriptableObject`
7. Open `Assets/_Project/Scenes/Level03.unity` to inspect the current Steam Scattergun pickup and steampunk boilerheart work.
8. Use the Hierarchy search box for object names such as:
   - `Steam Scattergun`
   - `Pickup`
   - `Scrapper`
   - `Lancer`
   - `Bulwark`
   - `Warden`
   - `Valve`
   - `Gauge`
9. Select an object in the Hierarchy, then press `F` in the Scene view to frame it.
10. Press Play to see materials, lights, VFX, and gameplay objects together in the Game view.

## What The Preview Sheets Do Not Show

The generated preview sheets are useful for quick inspection, but they are not a full render of the game.

They do not show:

- actual scene lighting
- material smoothness, metallic response, or shader behavior
- particle motion, steam, muzzle flashes, or hit effects
- scale and placement inside levels
- player camera framing
- animated machinery or enemies

For those, Unity's Scene view and Game view are the source of truth.

## Practical Recommendation

Use `Documentation/AssetPreviews` for quick visual browsing, then use Unity for final inspection. The project is currently still prototype-heavy: materials, generated textures, concept art, scenes, and procedural geometry are real Unity assets, but the final AAA-style imported model and texture pack has not been built yet.
